Импорт списка пробежавших



Разделитель колонок — точка с запятой (;). Колонки: first_name, last_name (обязательны, раздельно — чтобы не гадать, в каком порядке в файле написаны имя и фамилия), email, phone, result (не обязательны). По умолчанию каждая строка получает статус «пробежал»; если в result стоит 0 — статус будет DNF (бежал, но не финишировал). Повторный импорт того же файла безопасен, дубли по имени пропускаются.

Email сопоставляется с зарегистрированным аккаунтом; если email нет или он не совпал — по имени/фамилии ищется уже существующий гостевой профиль (в том числе уже объединённый с реальным аккаунтом — тогда запись уходит сразу на него), и только если совпадения нет вообще — заводится новый гостевой профиль.

Пример файла
first_name last_name email phone result
Иван Петров ivan@example.com +79991234567 1
Мария Сидорова 0
Алексей Кузнецов

Мария не финишировала (result = 0), Алексей — без email/телефона/ result, что по умолчанию тоже считается «пробежал». То же самое в виде CSV-текста (разделитель ;): 

first_name;last_name;email;phone;result
Иван;Петров;ivan@example.com;+79991234567;1
Мария;Сидорова;;;0
Алексей;Кузнецов;;;
